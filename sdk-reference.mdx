The Wraith SDK instruments your backend to capture traces — complete records of HTTP requests, database queries, outbound calls, and errors.

## Fastify tracing plugin

Register the plugin **before** your route definitions.

```typescript
import Fastify from 'fastify';
import { fastifyTracing } from '@runtime-digital-twin/sdk';

const fastify = Fastify();

await fastify.register(fastifyTracing, {
  serviceName: 'my-service',
  serviceVersion: '1.0.0',
  environment: 'production',
  traceDir: './traces',
  uploadUrl: `${process.env.WRAITH_API_URL}/api/traces/ingest`,
  apiKey: process.env.WRAITH_API_KEY,
  uploadOnComplete: true,
});
```

### All options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `serviceName` | `string` | **required** | Service identifier. Must match your monitoring tool. |
| `serviceVersion` | `string` | — | Your service version |
| `environment` | `string` | — | Environment name (production, staging, etc.) |
| `traceDir` | `string` | `./traces` | Where trace bundles are written locally |
| `headerAllowlist` | `string[]` | `[]` | Additional headers to capture |
| `maxBodySize` | `number` | `10485760` | Max request/response body size (bytes) |
| `enabled` | `boolean` | `true` | Enable/disable tracing |
| `traceSampleRate` | `number` | `1` | Fraction of requests to trace (0-1) |
| `alwaysTraceErrors` | `boolean` | `true` | Always trace requests that error, even if not sampled |
| `uploadUrl` | `string` | — | Wraith ingest URL |
| `apiKey` | `string` | — | API key from Settings > API Keys |
| `uploadOnComplete` | `boolean` | `true` | Auto-upload after each request |

### Sampling strategy

<Info>
  `alwaysTraceErrors: true` (the default) ensures failing requests are always captured, regardless of sample rate.
</Info>

| Traffic | Recommended `traceSampleRate` |
|---------|-------------------------------|
| < 100 req/min | `1` (100%) |
| 100–1000 req/min | `0.1` – `0.5` |
| > 1000 req/min | `0.01` – `0.1` |

Uploads happen asynchronously and do not slow down your response time. If the ingest endpoint returns `429`, the SDK retries with exponential backoff.

## Database wrapper

Wraps your PostgreSQL client to capture all queries in traces. Drop-in replacement.

```typescript
import { createWrappedDbClient } from '@runtime-digital-twin/sdk';
import { Pool } from 'pg';

const db = await createWrappedDbClient({
  realClient: new Pool({ connectionString: process.env.DATABASE_URL }),
});

const result = await db.query('SELECT * FROM users WHERE id = $1', [userId]);
```

For every query, the SDK records the SQL statement, parameters, result set, execution time, and errors.

## HTTP client wrapper

Captures outbound HTTP calls for complete trace coverage.

```typescript
import { wrapFetch } from '@runtime-digital-twin/sdk';

const wrappedFetch = wrapFetch(fetch);

const response = await wrappedFetch('https://api.stripe.com/v1/charges', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${process.env.STRIPE_KEY}` },
  body: 'amount=999&currency=usd',
});
```

<Warning>
  Without wrapping outbound calls, Wraith can't see the full picture of what happened during a request. Wrap `fetch` for all services that call external APIs.
</Warning>

## Trace context

The SDK automatically generates correlation IDs for cross-service tracing.

```typescript
import { getTraceContext, setTraceContext } from '@runtime-digital-twin/sdk';

const context = getTraceContext();
setTraceContext({ traceId: 'custom-id', spanId: 'custom-span' });
```

When your service calls another instrumented service, the SDK propagates the correlation ID via HTTP headers automatically.

## Environment variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `WRAITH_API_URL` | For upload | — | Your Wraith instance URL |
| `WRAITH_API_KEY` | For upload | — | API key from Settings > API Keys |
| `RDT_TRACE_DIR` | No | `./traces` | Trace storage directory |

## Service name consistency

<Warning>
  The `serviceName` must be identical across all systems. Mismatches prevent Wraith from correlating alerts to traces.
</Warning>

| System | Where to set |
|--------|-------------|
| **SDK** | `serviceName` in `fastifyTracing` |
| **Datadog** | Service tag on monitors |
| **CloudWatch** | Log group or alarm dimension |
| **Sentry** | `serverName` in Sentry config |

## Complete example

```typescript
import Fastify from 'fastify';
import { fastifyTracing, createWrappedDbClient, wrapFetch } from '@runtime-digital-twin/sdk';
import { Pool } from 'pg';

async function start() {
  const fastify = Fastify({ logger: true });

  await fastify.register(fastifyTracing, {
    serviceName: 'user-api',
    serviceVersion: '1.0.0',
    environment: process.env.NODE_ENV || 'production',
    traceDir: './traces',
    uploadUrl: `${process.env.WRAITH_API_URL}/api/traces/ingest`,
    apiKey: process.env.WRAITH_API_KEY,
  });

  const db = await createWrappedDbClient({
    realClient: new Pool({ connectionString: process.env.DATABASE_URL }),
  });

  const fetchApi = wrapFetch(fetch);

  fastify.get('/api/users/:id', async (request, reply) => {
    const { id } = request.params as { id: string };
    const result = await db.query('SELECT * FROM users WHERE id = $1', [id]);
    if (result.rows.length === 0) return reply.code(404).send({ error: 'Not found' });
    return result.rows[0];
  });

  await fastify.listen({ port: 3000, host: '0.0.0.0' });
}

start().catch(console.error);
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Traces not being created">
    Check that `traceDir` is writable, `enabled` is `true` (default), and review application logs for SDK errors.
  </Accordion>
  <Accordion title="Traces not uploading">
    Verify `WRAITH_API_URL` and `WRAITH_API_KEY` are set. Check that your deployment can reach the Wraith URL. If you see `429` errors, reduce `traceSampleRate`.
  </Accordion>
  <Accordion title="Database queries not captured">
    Call `createWrappedDbClient` before using the client in any route. Verify `DATABASE_URL` is set.
  </Accordion>
  <Accordion title="High memory usage">
    Reduce `maxBodySize` if payloads are large. Enable sampling with `traceSampleRate` below 1.
  </Accordion>
</AccordionGroup>
