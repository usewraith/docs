---
title: "Alert Integrations"
description: "Connect your monitoring tools to Wraith via webhooks. Supports Datadog, CloudWatch, Sentry, PagerDuty, Grafana, Opsgenie, and Bugsnag."
---

Wraith receives alerts from your monitoring tools via webhooks. Each provider has a dedicated endpoint that normalizes the payload and feeds it into the incident pipeline.

## Webhook URLs

All webhook URLs follow the pattern:

```
https://<your-wraith-url>/api/alerts/<provider>/<your-org-id>
```

<Warning>
  Get the exact URL from **Settings > Integrations** in your dashboard. The organization ID is embedded in the path — do not modify it.
</Warning>

All providers (except CloudWatch) authenticate with the header `x-wraith-webhook-secret`.

## Datadog

<Steps>
  <Step title="Get webhook URL">
    In Wraith, go to **Settings > Integrations > Datadog**. Copy the **Webhook URL** and **Webhook Secret**.
  </Step>
  <Step title="Create webhook in Datadog">
    Go to **Integrations > Webhooks > New Webhook**.

    - **Name**: `Wraith On-Call`
    - **URL**: Paste the webhook URL from Wraith
    - **Custom Headers**: `x-wraith-webhook-secret: <your-secret>`
  </Step>
  <Step title="Add to monitors">
    Add `@webhook-Wraith On-Call` to your Datadog monitor notification fields.
  </Step>
</Steps>

**Test it:**
```bash
curl -X POST https://<your-wraith-url>/api/alerts/datadog/<your-org-id> \
  -H "Content-Type: application/json" \
  -H "x-wraith-webhook-secret: <your-secret>" \
  -d '{"event":{"title":"Test Alert","text":"Testing Wraith","alert_type":"error","service":"user-api","environment":"production"}}'
```

## AWS CloudWatch

<Steps>
  <Step title="Get webhook URL">
    In Wraith, go to **Settings > Integrations > CloudWatch**. Copy the **Webhook URL**.
  </Step>
  <Step title="Create SNS topic">
    ```bash
    aws sns create-topic --name wraith-alerts
    ```
  </Step>
  <Step title="Subscribe Wraith endpoint">
    ```bash
    aws sns subscribe \
      --topic-arn arn:aws:sns:<region>:<account-id>:wraith-alerts \
      --protocol https \
      --notification-endpoint <webhook-url-from-wraith>
    ```
    Wraith auto-confirms the SNS subscription.
  </Step>
  <Step title="Attach alarms">
    Point your CloudWatch alarms to the SNS topic.
  </Step>
</Steps>

## Sentry

<Steps>
  <Step title="Get webhook URL">
    In Wraith, go to **Settings > Integrations > Sentry**. Copy the **Webhook URL** and **Secret**.
  </Step>
  <Step title="Configure Sentry">
    Go to **Settings > Integrations > Webhooks**. Add the URL and configure alert rules to send to it.
  </Step>
</Steps>

<Info>
  Ensure Sentry's `serverName` matches your Wraith `serviceName`:

  ```javascript
  Sentry.init({ dsn: '...', serverName: 'user-api' });
  ```
</Info>

## PagerDuty

<Steps>
  <Step title="Get webhook URL">
    In Wraith, go to **Settings > Integrations > PagerDuty**. Copy the URL and secret.
  </Step>
  <Step title="Add integration">
    In PagerDuty, go to **Services > [Your Service] > Integrations > Add Integration**. Select **Generic Webhook (v3)** and set the URL.
  </Step>
</Steps>

## Grafana

<Steps>
  <Step title="Get webhook URL">
    Copy the URL and secret from **Settings > Integrations > Grafana**.
  </Step>
  <Step title="Create contact point">
    In Grafana, go to **Alerting > Contact Points > New Contact Point**. Set type to **Webhook**, paste the URL, and add the header `x-wraith-webhook-secret: <your-secret>`.
  </Step>
  <Step title="Route alerts">
    Route your alert rules to this contact point.
  </Step>
</Steps>

## Opsgenie

1. Copy URL and secret from **Settings > Integrations > Opsgenie**.
2. In Opsgenie, go to **Teams > [Your Team] > Integrations > Add Integration > Webhook**.
3. Set the URL and add the header. Configure alert policies to forward alerts.

## Bugsnag

1. Copy URL and secret from **Settings > Integrations > Bugsnag**.
2. In Bugsnag, go to **Project Settings > Integrations > Webhook**.
3. Set the URL, add the header, and configure which events trigger it.

## Multiple alert sources

You can connect multiple monitoring tools at once. Wraith handles deduplication across sources — if two tools fire for the same issue, Wraith creates a single incident.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Alert not received">
    Copy the webhook URL directly from **Settings > Integrations**. Verify the secret matches. Check that your monitoring tool can reach the Wraith URL.
  </Accordion>
  <Accordion title="Alert received but no incident">
    Wraith may have classified it as noise. Check the audit log at `GET /api/layer0/audit`. The alert must include a `service` name.
  </Accordion>
  <Accordion title="Service name mismatch">
    The most common issue. The service name must be identical across SDK (`serviceName`), monitoring tool (service tag), and Wraith.
  </Accordion>
</AccordionGroup>
