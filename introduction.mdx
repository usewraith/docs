## Set up your editor

<Tip>
  Using an AI coding tool? Get interactive, guided setup in seconds.
</Tip>

<Tabs>
  <Tab title="Claude Code">
    Run in your project directory:

    ```bash
    npm install @runtime-digital-twin/sdk && claude
    ```

    Then paste this prompt:

    ```text
    Set up Wraith On-Call Engineer for this project. Read package.json, find
    my server entry point, and walk me through adding fastifyTracing,
    createWrappedDbClient, and wrapFetch from @runtime-digital-twin/sdk.
    Ask me one question at a time.
    ```

    Connect Wraith's MCP server for incident investigation:

    ```bash
    claude mcp add wraith --transport http https://<your-wraith-url>/api/mcp \
      -- --header "Authorization: Bearer <your-api-key>"
    ```
  </Tab>
  <Tab title="Cursor">
    Install the SDK:

    ```bash
    npm install @runtime-digital-twin/sdk
    ```

    Open Cursor Chat (`Cmd+L`) and paste:

    ```text
    Set up Wraith On-Call Engineer for this project. Read package.json, find
    my server entry point, and walk me through adding fastifyTracing,
    createWrappedDbClient, and wrapFetch from @runtime-digital-twin/sdk.
    Ask me one question at a time.
    ```

    Connect MCP in **Settings > MCP Servers > Add**:

    ```json
    {
      "wraith": {
        "url": "https://<your-wraith-url>/api/mcp",
        "type": "http",
        "headers": { "Authorization": "Bearer <your-api-key>" }
      }
    }
    ```
  </Tab>
</Tabs>

## How it works

<Steps>
  <Step title="Alert fires">
    Your monitoring tool (Datadog, CloudWatch, Sentry, PagerDuty, Grafana, Opsgenie, or Bugsnag) sends a webhook to Wraith.
  </Step>
  <Step title="AI triage">
    Wraith filters noise, detects duplicates, and analyzes service dependencies to determine if the alert is real.
  </Step>
  <Step title="Automated investigation">
    Selects the most relevant traces, runs AI-powered verification, and identifies the root cause.
  </Step>
  <Step title="You get notified">
    Via Slack, PagerDuty, or the dashboard — with full evidence and root cause analysis.
  </Step>
  <Step title="Optional AutoFix">
    Generates a verified code patch and opens a GitHub PR for review.
  </Step>
</Steps>

## MCP tools

Once you connect the MCP server, these tools are available in your editor:

| Tool | Description |
|------|-------------|
| `search_incidents` | Find incidents by service, status, or time range |
| `get_investigation_detail` | Full evidence, context, and fix jobs for an incident |
| `compare_incidents` | Side-by-side comparison of two incidents |
| `get_similar_fixes` | Past fixes that match this failure pattern |
| `get_verification_report` | Root cause, confidence score, and evidence |
