The Wraith API is a REST API served by the Next.js application.

## Authentication

### API Key

```
Authorization: Bearer <your-api-key>
```

Get your API key from **Settings > API Keys** in the dashboard.

### Session

Dashboard API calls use session cookies (handled automatically by the browser).

### Webhook

Alert endpoints use `x-wraith-webhook-secret: <your-secret>`.


## Base URL

```
https://<your-wraith-url>
```


## MCP Endpoint

```
POST /api/mcp
```

Model Context Protocol endpoint for IDE integrations (Cursor, Claude Code). Supports `initialize`, `tools/list`, and `tools/call` methods.

**Auth**: Bearer token (API key) or browser session.

**Available tools**: `search_incidents`, `get_investigation_detail`, `compare_incidents`, `list_traces_replay_status`, `get_fix_context_manifest`, `get_fix_context_section`, `get_similar_fixes`, `get_sandbox_status`, `get_sandbox_logs`, `get_verification_report`.


## Alert Ingestion

These endpoints use webhook secret authentication, not API keys.

### Datadog

```
POST /api/alerts/datadog/<orgId>
```

**Headers**: `x-wraith-webhook-secret: <your-secret>`

**Body**:
```json
{
  "event": {
    "title": "Alert title",
    "text": "Alert description",
    "alert_type": "error",
    "service": "my-service",
    "environment": "production"
  }
}
```

### CloudWatch

```
POST /api/alerts/cloudwatch/<orgId>
```

**Body**: SNS notification message (auto-verified).

### Sentry

### PagerDuty

## Traces

### Ingest a Trace

```
POST /api/traces/ingest
```

**Auth**: API key. **Body**: Trace bundle (generated by SDK).

### List Traces

```
GET /api/traces
```

**Query**: `serviceName`, `environment`, `limit`, `offset`

### Get Trace Detail

```
GET /api/traces/<id>
```

### Get Trace Bug Signals

```
GET /api/traces/<id>/bug-signals
```

### Get Trace Status

```
GET /api/traces/<id>/status
```

### Download Trace

```
GET /api/traces/<id>/download
```


## Incidents

### List Incidents

```
GET /api/incidents/list
```

**Query**: `status`, `service`, `limit`, `offset`

**Response**:
```json
{
  "incidents": [
    {
      "id": "inc_abc123",
      "title": "High Error Rate on user-api",
      "status": "ESCALATED",
      "service": "user-api",
      "confidence": 0.92,
      "createdAt": "2026-03-29T12:00:00Z"
    }
  ],
  "total": 45
}
```

### Get Incident Detail

```
GET /api/incidents/<id>
```

### Provide Feedback

```
POST /api/incidents/<id>/feedback
```

**Body**:
```json
{
  "verdictCorrect": true,
  "comment": "Root cause was correctly identified"
}
```

### Analyze Commits

```
POST /api/incidents/<id>/analyze-commits
```

Triggers on-demand commit analysis (bisection).

### Get Pipeline Progress

```
GET /api/incidents/<id>/pipeline-progress
```

Step-by-step pipeline logs with timing and status.

### Get Multi-Service Root Cause

```
GET /api/incidents/<id>/multi-service-rca
```

### Get Session Recording

```
GET /api/incidents/<id>/session-replay
```

rrweb DOM recording data for frontend incidents.

## Error Format

```json
{
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": {}
}
```

| Code | Meaning |
|------|---------|
| `400` | Bad request |
| `401` | Unauthorized |
| `403` | Forbidden |
| `404` | Not found |
| `429` | Rate limited (check `Retry-After` header) |
| `500` | Internal error |
