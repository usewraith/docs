After you complete this guide, you'll have traces flowing from your app and alerts creating incidents in Wraith.

## Before you begin

You need a Wraith account and access to the dashboard. Go to **Settings > API Keys** and create an API key — you'll need it for trace uploads.

<Info>
  Your API key starts being shown only once. Store it securely.
</Info>

## Install the SDK

<CodeGroup>

```bash npm
npm install @runtime-digital-twin/sdk
```

```bash pnpm
pnpm add @runtime-digital-twin/sdk
```

```bash yarn
yarn add @runtime-digital-twin/sdk
```

</CodeGroup>

## Add tracing to your server

Add the Fastify plugin **before** your route definitions:

```typescript
import Fastify from 'fastify';
import { fastifyTracing } from '@runtime-digital-twin/sdk';

const fastify = Fastify();

await fastify.register(fastifyTracing, {
  serviceName: 'my-api-service',
  serviceVersion: '1.0.0',
  environment: process.env.NODE_ENV || 'production',
  traceDir: './traces',
  uploadUrl: `${process.env.WRAITH_API_URL}/api/traces/ingest`,
  apiKey: process.env.WRAITH_API_KEY,
  uploadOnComplete: true,
});
```

<Warning>
  The `serviceName` must match what your monitoring tool uses. This is how Wraith correlates alerts to traces.
</Warning>

## Wrap your database

```typescript
import { createWrappedDbClient } from '@runtime-digital-twin/sdk';
import { Pool } from 'pg';

const db = await createWrappedDbClient({
  realClient: new Pool({ connectionString: process.env.DATABASE_URL }),
});

// Use db.query() as normal
```

## Wrap outbound HTTP calls

If your service calls external APIs, wrap `fetch` for full trace coverage:

```typescript
import { wrapFetch } from '@runtime-digital-twin/sdk';

const wrappedFetch = wrapFetch(fetch);
// Use wrappedFetch instead of fetch
```

<Tip>
  Without wrapping outbound calls, Wraith can't see the full picture of what happened during a request.
</Tip>

## Set environment variables

Add to your `.env` or deployment config:

```bash
WRAITH_API_URL=https://<your-wraith-url>
WRAITH_API_KEY=<your-api-key>
```

## Connect your monitoring tool

Get the webhook URL from **Settings > Integrations** in your Wraith dashboard.

<Tabs>
  <Tab title="Datadog">
    <Steps>
      <Step title="Get webhook URL">
        In Wraith, go to **Settings > Integrations > Datadog** and copy the webhook URL and secret.
      </Step>
      <Step title="Create webhook in Datadog">
        Go to **Integrations > Webhooks > New Webhook**. Paste the URL and add the header `x-wraith-webhook-secret: <your-secret>`.
      </Step>
      <Step title="Add to monitors">
        Add `@webhook-<name>` to your Datadog monitor notification fields.
      </Step>
    </Steps>
  </Tab>
  <Tab title="CloudWatch">
    <Steps>
      <Step title="Create SNS topic">
        ```bash
        aws sns create-topic --name wraith-alerts
        ```
      </Step>
      <Step title="Subscribe Wraith endpoint">
        ```bash
        aws sns subscribe \
          --topic-arn arn:aws:sns:<region>:<account>:wraith-alerts \
          --protocol https \
          --notification-endpoint <webhook-url-from-wraith>
        ```
      </Step>
      <Step title="Attach alarms">
        Point your CloudWatch alarms to the SNS topic.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Sentry">
    <Steps>
      <Step title="Get webhook URL">
        In Wraith, go to **Settings > Integrations > Sentry** and copy the URL and secret.
      </Step>
      <Step title="Configure Sentry">
        Go to **Settings > Integrations > Webhooks** and add the URL. Configure alert rules to send to the webhook.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Others">
    PagerDuty, Grafana, Opsgenie, and Bugsnag all follow the same pattern — get the webhook URL from **Settings > Integrations** and configure your tool to send alerts to it.

    See [Alert Integrations](/alert-integrations) for detailed per-provider instructions.
  </Tab>
</Tabs>

## Test it

Send a test alert to verify everything is connected:

```bash
curl -X POST https://<your-wraith-url>/api/alerts/datadog/<your-org-id> \
  -H "Content-Type: application/json" \
  -H "x-wraith-webhook-secret: <your-secret>" \
  -d '{"event":{"title":"Test Alert","text":"Testing Wraith","alert_type":"error","service":"my-api-service","environment":"production"}}'
```

Open **Incidents** in your Wraith dashboard — you should see the test incident within seconds.

<AccordionGroup>
  <Accordion title="Incident stuck at RECEIVED?">
    This means the pipeline didn't start. Check that your alert's `service` name matches the `serviceName` in your SDK config. If the pipeline still doesn't run, contact your Wraith administrator to verify the workflow engine is configured.
  </Accordion>
  <Accordion title="Webhook returning errors?">
    Copy the URL directly from **Settings > Integrations** — don't modify the org ID in the path. Verify the webhook secret matches exactly.
  </Accordion>
</AccordionGroup>
