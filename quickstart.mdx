# Quickstart

Get Wraith connected to your app and monitoring tool in under 5 minutes.

## 1. Install the SDK

```bash
npm install @runtime-digital-twin/sdk
```

## 2. Add tracing to your server

Add the Fastify plugin **before** your route definitions:

```typescript
import Fastify from 'fastify';
import { fastifyTracing } from '@runtime-digital-twin/sdk';

const fastify = Fastify();

await fastify.register(fastifyTracing, {
  serviceName: 'my-api-service',
  serviceVersion: '1.0.0',
  environment: process.env.NODE_ENV || 'production',
  traceDir: './traces',
  uploadUrl: `${process.env.WRAITH_API_URL}/api/traces/ingest`,
  apiKey: process.env.WRAITH_API_KEY,
  uploadOnComplete: true,
});
```

<Info>
  The `serviceName` must match what your monitoring tool uses. This is how Wraith correlates alerts to traces.
</Info>

## 3. Wrap your database

```typescript
import { createWrappedDbClient } from '@runtime-digital-twin/sdk';
import { Pool } from 'pg';

const db = await createWrappedDbClient({
  realClient: new Pool({ connectionString: process.env.DATABASE_URL }),
});
```

## 4. Wrap outbound HTTP (optional)

If your service calls other APIs:

```typescript
import { wrapFetch } from '@runtime-digital-twin/sdk';

const wrappedFetch = wrapFetch(fetch);
```

## 5. Set environment variables

```bash
WRAITH_API_URL=https://<your-wraith-url>
WRAITH_API_KEY=<your-api-key>
```

<Tip>
  Get your API key from **Settings > API Keys** in the Wraith dashboard.
</Tip>

## 6. Connect your monitoring tool

Get your webhook URL from **Settings > Integrations** in the Wraith dashboard, then configure your monitoring tool:

<Tabs>
  <Tab title="Datadog">
    1. Go to **Integrations > Webhooks > New Webhook**
    2. **URL**: paste the webhook URL from Wraith
    3. **Headers**: `x-wraith-webhook-secret: <your-secret>`
    4. Add `@webhook-<name>` to your monitor notifications
  </Tab>
  <Tab title="CloudWatch">
    ```bash
    aws sns create-topic --name wraith-alerts

    aws sns subscribe \
      --topic-arn arn:aws:sns:<region>:<account>:wraith-alerts \
      --protocol https \
      --notification-endpoint <webhook-url>
    ```
    Attach your CloudWatch alarms to the topic.
  </Tab>
  <Tab title="Sentry">
    1. Go to **Settings > Integrations > Webhooks**
    2. Add the webhook URL from Wraith
    3. Configure alert rules to send to the webhook
  </Tab>
  <Tab title="Others">
    PagerDuty, Grafana, Opsgenie, and Bugsnag all follow the same pattern.
    See [Alert Integrations](/alert-integrations) for detailed setup.
  </Tab>
</Tabs>

## 7. Test it

```bash
curl -X POST https://<your-wraith-url>/api/alerts/datadog/<your-org-id> \
  -H "Content-Type: application/json" \
  -H "x-wraith-webhook-secret: <your-secret>" \
  -d '{"event":{"title":"Test Alert","text":"Testing Wraith","alert_type":"error","service":"my-api-service","environment":"production"}}'
```

Open **Incidents** in your Wraith dashboard — you should see the test incident.
